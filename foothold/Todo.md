# Tackle First

- [x] **Turn order fix** - Balance fix: first-mover imbalance - player 1 (always moves first each round) was winning 66.8% vs player 2's 33.1% over 5000 AI-vs-AI games. Considered alternating who moves first, but that would read as "the enemy went twice in a row" at the round boundary - kept turn order fixed and compensated player 2 with a starting-resource bump instead (`P2_START_BONUS = { wood: 14, gold: 14, stone: 7 }`, tuned against the harness). Invisible to the player (AI's stockpile is never shown in the UI). Re-ran at 20,000 games: 50.7%/49.3%.
- [x] **Mobile tooltip** - Mobile tooltip preview problem - the tooltip was wired to `pointerover`/`pointerout`, a hover state that doesn't exist on touch, so `pointerdown` fired the move immediately with no preview step. Fixed with tap-to-select/tap-again-to-confirm: `pointerdown` now routes through `onTilePointerDown`, which only commits (`onTileTap`) if the tapped tile is already `this.selected`; otherwise it just calls `showTooltip` to preview and select it. `showTooltip`/`hideTooltip` now own `this.selected`, and a board-wide `pointerdown` listener clears it when the tap hits nothing (river/margins/HUD). On desktop this is invisible - hover already selects the tile before the click lands, so a single click still commits in one motion, same as before.
- [x] **PWA support** - PWA build for the web version - `manifest.webmanifest` + `sw.js` (cache-first service worker, offline + installable), wired into `index.html`, verified in-browser (SW registers/activates, cache holds all 24 precached files). Icons are a temporary placeholder (upscaled `watchtower.png`) - swap to the real files once the custom app icon task is done. CLAUDE.md's pre-push gate now has the `CACHE_VERSION` bump step.
- [x] **Total Victory** - Total Victory (fun polish) - capturing the enemy base while owning every non-river tile and having every held node upgraded now triggers "Complete Foothold! / Round N" / "The enemy has nothing left", a gold sunburst (pulsing ADD-blend glow + 10 rotating rays, sized off the canvas diagonal so it reaches the screen edges in both portrait and wide layouts), a camera flash/shake, and a unique "Grand Finale" fanfare (`sfx.js`: horn call, rising flourish, sustained chord, shimmering bells). Staged 10 fanfare candidates + 3 flare variants in `stage/total-victory.html` before picking. Tuned glow brightness down twice after playtest (alpha 1 -> 0.55 -> 0.28) since the ADD blend stacked much brighter than the mockup.

# Future Ideas

Ideas for future maps/levels, each its own board with its own terrain gimmick - not reskins of the river, and not required to keep a river at all (the river is just level 1's terrain, other levels can drop it or replace it with something else entirely):
- [ ] **Mountains level** - Mountains - a SOLID blocking terrain, zero crossings anywhere. Splits the board into disconnected pockets, so the opening becomes "which pocket did I land in" rather than a symmetric two-halves map. First pass (4 static-split ideas) was reviewed and rejected: "two separate board sections with little interaction... just luck of the draw for who gets more tiles on their side." Redesigned in `stage/mountain-pass.html` around making the wall itself a contested, active object instead of fixed geography - all 4 new ideas force mutual interaction every round: Avalanche Tug (both players spend stone to push a shared tug-of-war meter; whichever side wins, one wall tile crumbles into their pocket - repeats all match), The One Pass (a single scarce peak tile grants vision + discounted siege, but either side can contest and flip ownership at any time), Tunnel Race (both dig simultaneously on visible progress meters, can spend gold to sabotage the opponent's progress, first to break through picks the crossing point), Rockfall Events (telegraphed random 1-round wall openings both players are incentivized to rush at the same time, forcing a clash).
- [ ] **Fire/hell level** - Fire/hell level - hazard tiles (lava) that tax whoever holds them: an owned hazard tile periodically burns a bit of stockpile instead of producing income, so it's a high-risk/high-reward land grab (probably paired with a bonus resource on/near it) rather than free territory. The high risk or burn territories should have a visual flame indicator when it's burning your resources along with a unique sound. 4 ideas staged in `stage/lava-fire.html`: Volcano Meteor Strikes (reworked per feedback - the volcano now sits as a fixed 2x2 impassable tile cluster in the middle of the board, not off-screen scenery, and erupts on a real-time clock running independently of turn order for a different pacing pressure than the other levels; still launches meteors onto random tiles, igniting them for a round before they revert to reclaimable scorched rubble), Lava Snake (a Snake/Worm-style lava worm crawls a telegraphed path across the board, scorching whatever it currently occupies), Ember Node (the valuable-but-punishing resource idea - a high-income node that periodically flares and burns a chunk of your stockpile as a tax for holding it), Spreading Wildfire (fire ignites at a point and creeps tile-to-tile each round unless a player-held tile acts as a firebreak).
- [ ] **Forest level** - Forest level - fog of war: enemy tiles/nodes outside your vision radius are hidden until a tile of yours is adjacent to them. Changes the game from perfect-information to scouting-driven; the AI would need vision logic added too, not just new art. Visual direction picked: "C - Foliage Canopy" fog treatment staged in `stage/forest-fog.html` - a dark mossy forest-floor base under two slow-drifting grey-white mist layers, painted once across the whole board (not per-tile) so canopy blobs and mist banks read as continuous. Priciest of the 3 staged options to build well for real (would want a proper Aseprite/painted fog texture + sprite animation instead of CSS gradients). Reveal holes are soft-edged/blurred so newly-scouted ground fades in. 3 more mechanic ideas as tabs in the same file: Wildwalker Incursion (a neutral third force raids unscouted tiles at the fog's edge with brambles if you go too long without scouting - confirmed as the favorite of the extra ideas), Overgrowth Reclaim (unscouted tiles grow denser vines and get progressively more expensive to claim the longer they're ignored), Canopy Watch (replaced the original Stealth Strike idea, which didn't read clearly - a wandering owl perches on a fogged tile and grants bonus vision to whoever claims that specific tile, then moves on to a new tile, making vision itself a moving, contested resource).
- [ ] **Ocean level** - Ocean level - water covers most of the board and crossings (bridges, or a new build-a-bridge action) are scarce/expensive, so crossings themselves become the thing players fight over, instead of the open land they currently fight over on a river map. Limited/scarce resources on islands that you build between. First pass (board-wide sweeping waterline) was reviewed and rejected: the tide should be a dedicated, randomized shoreline zone, not a sweep across the whole board. Reworked in `stage/ocean-tides.html` so each variant now floods only 1-2 tiles at a time within a small fixed zone, leaving everything else untouched: Rolling Shoreline (a bounded band tucked in one corner, waterline moves 1 row in/out per cycle), Cove Sweep (a small L-shaped wedge in a corner, 1-2 tiles move at its outer edge), Tide Pools (2-3 small fixed pools, each capped to a 3x3 footprint so only the rim moves), Pincer Tide (two small strips near each player's home flank instead of a board-wide squeeze).

- [ ] **Level select screen** - New game flow: a level-select screen after Start, picking between the 5 levels (River + the 4 above). Each level shown as a row with a mechanic-highlighting thumbnail on the left and a title + one-line description on the right; locked/"Coming soon" rows for the 4 unbuilt levels. Staged in `stage/level-select.html`. Open question noted there: whether this replaces the current single Start button outright, or only appears once a second level actually ships.
