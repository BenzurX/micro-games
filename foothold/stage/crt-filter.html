If we ship one, how it would be built

  • Any curve (A+ / A++ / B) needs the GPU: the barrel bend is a per-pixel warp, so it has to be a small Phaser WebGL PostFX pipeline — barrel warp + scanlines + bloom + vignette in one fragment shader. Cheap on any phone GPU, but it is real engine code. The bezel + glare around it are just page CSS.
  • Flat A alone is nearly free: if you drop the curve, A is a fixed overlay div on #game (repeating-gradient scanlines + vignette) plus a one-line CSS filter for the glow — no engine changes. Adding the bend is what pulls in the shader.
  • Recommendation if adopted: go A+ or A++ (whichever curve you prefer) and make it a toggle in the upcoming settings screen, likely off by default so players opt in. The shader carries all three curve levels; the difference is one warp constant.
  • This page is design scratch under stage/ — nothing here is wired into the game.
